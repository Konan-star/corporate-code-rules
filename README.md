## 規約の構成

本規約は以下の原則に基づいて構成されています：

1. **セキュリティファースト**: すべての実装でセキュリティを最優先
2. **保守性の確保**: 将来の変更に対応しやすいコード
3. **一貫性**: チーム全体で統一されたコーディングスタイル
4. **パフォーマンス**: スケーラブルなアーキテクチャ

## 重要な注意事項

### 必須のセキュリティルール
- パスワードは必ず**bcrypt**でハッシュ化
- APIキーや認証情報の**ハードコーディング禁止**
- 個人情報（PII）の**ログ出力禁止**
- すべてのユーザー入力の**検証必須**

### プロセスルール
- 新規ライブラリ導入には**事前のセキュリティレビュー**が必要
- 本番環境へのデプロイは**2名以上のレビュー**が必須
- データベース変更には**マイグレーションスクリプト**必須

### 規約の更新手順

1. **Issue作成**: 変更提案の議論
2. **Pull Request**: 具体的な変更内容
3. **レビュー**: 最低2名の承認が必要
4. **マージ**: mainブランチへの統合
5. **通知**: 全開発者への周知
